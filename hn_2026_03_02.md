Title:  
Show HN: Turn any GitHub Markdown into a collaborative editor by replacing “g” with "tune"

Body:  
Hi everyone,

I love Markdown but collaboration is a pain. I always fall back to a Google docs to get to a state I like together, to than export it to markdown, copy it locally, commit it and update my repo. 

Why Google docs? MD misses annotations, discsusions, Richttext that everybody (also non techies) know how to work with...

Thats why I build colibri, a Google Docs-like experience for any GitHub Markdown files. Accesible for everyone With or without a github account. 

Just replace the g of github.com with tune "tuneithub.com". 

[https://github.com/Legit-Control/get-colibri](https://github.com/Legit-Control/get-colibri)  
->  
[https://tuneithub.com/Legit-Control/get-colibri](https://tuneithub.com/Legit-Control/get-colibri)

and you can start to collaborate - press open PR - to get your changes merged back in. 

You can edit, annotate, and discuss in real time on your Markdown documents.

Why we built it

We founded a completely different startup, but noticed again how painful it is to collaborate on Markdown. We kept switching back to Google Docs for async and live edits, which felt awkward because docs,rfc's and so on belong next to the code, not locked in Google Docs forever.

Colibri lets you collaborate where the Markdown lives,  in real time, with suggestions and annotations.

Limitations:

- Works only for public GitHub repositories for now
- Support for private repos coming soon

Feedback, I’d love:

- How do you currently collaborate on .md files?
- If you could add one feature to a Markdown editor like this, what would it be?
